# 任务树视图设计方案

## 概述
在任务列表页面增加"树视图"模式，展示项目 → 任务的父子层级关系。

## 页面布局

```
▼ 项目A
   ▼ 父任务A
      ├─ 子任务A1
      └─ 子任务A2
▶ 项目B
   ...
默认项目
   ▼ 未分类
      独立任务（无父任务）
```

## 功能设计

### 1. 视图切换
- 三视图：列表 / 看板 / 树
- 切换按钮：列表 | 看板 | 树

### 2. 树形结构
- 第一层：项目（项目名）
- 第二层：父任务
- 第三层及以下：子任务

### 3. 节点内容
- 只显示任务标题
- 状态用标题颜色区分（已完成=绿色，其他=默认色）

### 4. 交互
- 点击任务节点 → 跳转到任务详情页
- 支持展开/折叠

## 技术实现

### 组件
- 使用 NTree 组件（Naive UI）

### 数据
- 调用 `/api/tasks` 获取所有任务
- 前端按项目分组，构建树形结构
