# 前置任务功能设计方案

## 概述
实现任务依赖/前置任务功能，当前置任务未完成时阻塞子任务。

## 1. 数据库改动

### tasks 表增加字段
```sql
-- JSON 格式存储依赖的任务 ID 数组
ALTER TABLE tasks ADD COLUMN depends_on TEXT;
-- 存储格式: [1,2,3] 或 null
```

## 2. 后端 API 设计

### 现有接口修改
- `POST /api/tasks` - 支持 depends_on 字段
- `PUT /api/tasks/<id>` - 支持 depends_on 字段
- `GET /api/tasks` - 返回数据包含 depends_on
- `GET /api/tasks/<id>` - 返回数据包含 depends_on

### 状态自动更新逻辑
- **当任务状态变为「已完成」时**：
  - 检查是否有任务依赖它（depends_on 包含此任务 ID）
  - 如果有，自动将依赖任务状态从「阻塞」改为「待处理」
- **当设置 depends_on 时**：
  - 检查依赖的前置任务是否未完成
  - 如果是，自动将当前任务状态设为「阻塞」

## 3. 前端设计

### 任务编辑表单
- 新增「前置任务」下拉选择（多选）
- 可选同项目下的其他任务
- 显示已选择的前置任务名称

### 任务列表/看板
- 前置任务未完成时：
  - 状态显示为「阻塞」（红色）
  - 禁用状态切换按钮
  - 显示依赖图标或提示
- 前置任务已完成时：正常显示

### 字段映射
| 状态 | 说明 |
|------|------|
| 阻塞 | 前置任务未完成 |
| 待处理 | 前置任务已完成或无依赖 |
